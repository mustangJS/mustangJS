Mustang JS

A JavaScript Library meant to make modifying & binding to the DOM much easier and faster, and in an Object-Oriented way.
IE9+, FireFox 27+, Chrome 33+, Safari 6+ , & Opera 21+
Object Oriented Code & Syntax

Examples

See some quick and concise examples of Mustang in action.

Querying an Element

            
              var elem = new Mustang(".classname");

Or, say you need all elements with a class including 'classname'.

Querying Multiple Elements

            
              var elems = new Mustang(".classname");

              // this returns a new MustangCollection

MustangCollections

Mustang Collections aren't anything extra special, it's just an object that holds an array of Mustang objects, with a few special functions. The keys of the array are in the order in which they are found in the DOM. There are different functions used to find specific elements in these returned MustangCollections. You can see them here.

Getting / Setting Content of Mustang objects and MustangCollections

            
              var elem = new Mustang(".single");

              elem.setContent("New Content");

              // or query multiple elements, and reset the content of them all
              var coll = new Mustang(".multiple");

              coll.setContent("All elements will be replaced with this string");

If you've made it this far, take a look at the docs below. There's a lot more to Mustang than just querying elements!

Mustang Object

Details :

The original Mustang object. Runs mostly off of the Sizzle.js selector engine, but we added a couple of functions ourselves to make DOM manipulation pretty easy & fun to do. If your familiar with jQuery this should be a breeze, with a brief change in syntax. Pass a css selector to the constructor and you'll recieve a new Mustang Object.

Instantiating Mustang Object

                        
                          // instantiate new object
                          var example = new Mustang(".classname");

Methods Reference

Mustang.getContent()

@var
(none)
@return
mixed

Grab the content (innerHTML) of an element.

                                
                                  // instantiate new object
                                  var example = new Mustang(".classname");

                                  var cont = example.getContent();

                                  // you can also method chain like below

                                  var cont = new Mustang(".classname").getContent();

Mustang.setContent(mixed)

@var
string, int, primitive data types
@return
Mustang Object

Sets the content (innerHTML) of an element.

                                
                                  // instantiate new object
                                  var example = new Mustang(".classname");

                                  var content = "String";
                                  example.setContent(content);

                                  content = 5;
                                  example.setContent(content);

                                  // you can also method chain like below

                                  var example = new Mustang(".classname").setContent("New Content");

Mustang.getType()

@var
(none)
@return
String, either "block" or "form"

Returns the type of an element, if it is a form (input, textarea, etc) element or a block element (p, h1, etc). This function is used more internally, but if you find a use for it, go ahead!

                                
                                  // instantiate new object
                                  var example = new Mustang(".classname");

                                  // returns 'form' or 'block'
                                  example.getType();

                                  // could use via method chaining
                                  var type = new Mustang(".selector").getType();

Mustang.bindToVariable(mixed *optional)

@var
Mixed, initial value for var
@return
Relationship object

Returns an object with interface available for modifying the model or element to which this is binded.

                                
                                  // instantiate new object
                                  var example = new Mustang(".classname");

                                  // returns 'Relationship' object, passing in initial value
                                  example.bindToVar(10);

                                  // could use via method chaining, no initial value
                                  var type = new Mustang(".selector").bindToVariable();

Mustang.textCountdown(span, max)

@var
Span, string as a selector for another element, typically a span or p etc.
max (optional), the max amount of characters to represented in the countdown
@return
Mustang Object

This is a typical front end situation, shows users how many characters are remaining in the current input/textarea. If no max parameter is passed, then the MAXLENGTH attribute of the input this is called on will be used. If no MAXLENGTH exists, unexpected behavior may occur. Only stops user's ability to type if MAXLENGTH is set within the HTML document.

                                
                                  // query an input element
                                  var example = new Mustang(".textCountdownExample");

                                  // example, portrayed below code
                                  example.textCountdown("#counter", 25);

Mustang.click(func)

@var
func
@return
Mustang Object

If you've used jQuery, this does the exact same thing. Pass a function to be called on the click of the current mustang element. You can either pass a predefined function as a parameter, or create a closure within the context that you call .click()! Either works, but beware of using `this`, as it does not point to the current MustangObject when used in a closure!

                                
                                  // instantiate new object
                                  var ex = new Mustang(".classname");

                                  ex.click(function(){
                                      console.log("clicking now");
                                  });

                                  function handler()
                                  {
                                      console.log("MustangJS is badass!");
                                  }

                                  ex.click(handler);

                                  ex.click(function(){
                                      // do not reference `this` in here! 
                                      // if need access to current element, just use the variable name
                                      ex.addClass("whatever");
                                  });

Mustang.type(func)

@var
func
@return
Mustang Object

This is practically the same as the Mustang.click() function. Just pass a method to be called when the current MustangObject is typed/keydown'd! By the way, this must be an input or textarea, so try not to use this method on non-form elements.

                                
                                  // instantiate new object
                                  var ex = new Mustang(".classname");

                                  ex.type(function(){
                                      console.log("typing now");
                                  });

                                  function handler()
                                  {
                                      console.log("MustangJS is badass!");
                                  }

                                  ex.type(handler);

                                  ex.type(function(){
                                      // do not reference `this` in here! 
                                      // if need access to current element, just use the variable name
                                      console.log(ex.getValue());
                                  });

MustangCollection Object

Details :

A MustangCollection is nothing more than an aggregated group of Mustang objects, returned when a query from the user returns more than one element that matches it. Nothing different needs to be done by the progammer, just access multiple elements the same way you would a single element, with a call to the Mustang class.

Instantiating MustangCollection Object

                        
                          // instantiate new object
                          var example = new Mustang(".multiple-elems");

Methods Reference

MustangCollection.getContent()

@var
(none)
@return
Array []

Grab the content (innerHTML) of each element. Returns an array of contents, in order that elements are found in page.

                            
                              // instantiate new object
                              var example = new Mustang(".classname");

                              var content = example.getContent();

                              // you can also method chain like below

                              var content = new Mustang(".classname").getContent();

MustangCollection.setContent()

@var
mixed
@return
(none)

Sets the content (innerHTML) of each element. Returns an array of contents, in order that elements are found in page. If given a string or value, all elements value will be set to that argument, but if given array, each element in collection parallel to element in array argument will be set.

                            
                              // instantiate new object
                              var example = new Mustang(".classname");

                              var content = example.setContent("All elems set to this string");

                              // array should have same number of keys as elements 
                              //in MustangCollection, else error will be thrown

                              var content = new Mustang(".classname").setContent(
                              ['each', 'element', 'has', 'different', 'value']
                              );

MustangCollection.find()

@var
int i
@return
Mustang object

Returns the Mustang Object present in the MustangCollection at position i. Access to elements is parallel to that of an array (i.e. the first element in a MustangCollection is at .find(0)).

                              var example = new Mustang(".classname");

                              // returns first element
                              var first = example.find(0);

MustangCollection.getType()

@var
@return
Array []

This is used more internally, but feel welcome to make use of it however you please. Returns an array of strings, representing each element in the MustangCollection. The two possible values are "block" and "form", depending on if element is a block element or an input/form element of any type.

                            
                              // instantiate new object
                              var example = new Mustang(".classname");

                              // returns first element
                              var first = example.getType();

                              // an example of a returned value
                              ['form', 'form', 'block', 'form']

MustangCollection.bindAll()

@var
(optional) vals[] - Array
@return
CollectionRelationship object

Ahh, getting down to the cool stuff. This binds all the elements belonging to a MustangCollection to a CollectionRelationship object. Arguments consists of an optional array of values to pass, which act as the initial values/contents for the elements. Notice if the element is a block element, it's content(innerHTML) will be set to the corresponding value, whereas a form element will have it's value set to the corresponding value from the vals argument. If no vals argument is passed, the value/content it is currently at the time of this function call will act as the initial values.

                            
                              // instantiate new object
                              var example = new Mustang(".classname");

                              // pass no vals argument
                              var rel = example.bindAll();

                              // pass array as argument
                              var rel = example.bindAll(
                              ['first value', 'second value', 'etc.']
                              );

MustangCollection.bindForm()

@var
(optional) vals{} - Object
@return
CollectionRelationship object

This function is especially cool, & similiar to the bindAll() function. Basically, the difference is that this function expects to be called on a form, (notice when you query a form, the form itself is not returned, but all of the elements inside the form) & it expects a JavaScript/JSON object with properties corresponding to the 'names' of the inputs and elements in the form. Every property in the argument, will be the initial value for the element in the form with the same name. As with bindAll(), the argument is optional and if one is not passed then the initial values are what they were at the time of the function call to bindForm(). See an example below.

                            
                              // instantiate new object
                              var example = new Mustang(".classname");

                              // pass no vals argument, binds all elems in form 
                              // excluding block elements and submit buttons
                              var rel = example.bindForm();

                              // pass object as argument
                              var rel = example.bindAll(
                              {
                                "nameOfInput" : "corresponding value",
                                "nextInputName" : 4
                              }
                              );

In this example of code, the input with the name of "nameOfInput" will be set to "corresponding value", and the input with the name of "nextInputName" will be set to 4. This is good for populating a form for an existing user that might want to edit some of their account settings, because you can grab the existing info, and pass it a MustangCollection caught by querying a form, and the form is populated with their existing info.

MustangCollection.getOdds()

@var
@return
MustangCollection object

This function returns a MustangCollection consisting of all the odd members from the previous MustangCollection. Differs from .find() in that the evens/odds are represented by what the user sees, and not accessed like an array. Therefore the first element on the screen that a user sees, would be considered odd, even though in the original MustangCollection it would be found at .find(0), which is technically an even number.

                            
                              // instantiate new object
                              var example = new Mustang(".classname");

                              // get odds 
                              var rel = example.getOdds();

Relationship Object

Details :

A relationship object represents a binding of data between an HTML element and a JavaScript variable. For the purpose of general use, a relationship object merely binds a property of a JavaScript object to the element's value/content, depending on the type of the variable. If the element given the relationship is an input, the value is stored, whereas if the element is a block element, the content(innerHTML) is stored. Don't worry about this though, the logic is abstracted away so that you don't have to worry about that, Mustang makes the proper decision for you based on the type of the element (block or form).

Instantiating Relationship Object

                        
                          // instantiate new Mustang object
                          var example = new Mustang(".single-elem");

                          // now bind it to a JavaScript variable, 
                          // and store the resulting object into a variable

                          var result = example.bindToVariable();

There are now multiple methods you can use to interact with the binding stored in result.

Methods Reference

@return

                              var example = new Mustang(".classname");

                              var rel = example.bindToVariable();

@var

@return

                              var example = new Mustang(".classname");

@var
(none)
@return
JSON Object

CollectionRelationship Object

Details :

A CollectionRelationship merely holds a list of Relationships between elements and values.

Instantiating Relationship Object

                        
                          // instantiate new Mustang object
                          var example = new Mustang(".multiple-elem");

                          // now bind it to a JavaScript variable, 
                          // and store the resulting object into a variable

                          var result = example.bindAll();

There are now multiple methods you can use to interact with the binding stored in result.

Methods Reference

Relationship.atIndex()

@var
int i
@return
Relationship object

Returns the Relationship object at the index of i. Access these like an array, starting at 0.

                            
                              // instantiate new object
                              var example = new Mustang(".multiple");

                              var rel = example.bindAll();
                              rel.atIndex(5).set("New value for model");
                              // notice we method chained to the returned Relationship from atIndex()

CollectionRelationship.toJson()

@var
(none)
@return
JSON Objects (Array)

Returns an array of properly formatted JSON objects, representing all of the elements in the CollectionRelationship object.

                            
                              // instantiate new object
                              var example = new Mustang(".multiple");

                              var rel = example.bindAll();
                              rel.toJson();

FormRelationship Object

Details :

A FormRelationship is a list of Relationships binded to the elements of a form.

Instantiating FormRelationship Object

                        
                          // instantiate new Mustang object
                          var example = new Mustang(".form-class");

                          var result = example.bindForm();

There is some interesting functionality happening under the hood of this object. Upon being instantiated, an argument can optionally be passed, representing a set of data that should be JSON compliant. Basically, the constructor assumes that the property names in the JSON object correspond to the names of the inputs within the form being binded, and can therefore be set initial values. Another exmaple is below.

                        
                          // say you hit a database, and got a result for a user
                          // and that result looks like the JSON obj below
                          var data = {
                            "name" : "Colby Perriwinkle",
                            "age"  : "120",
                            "fav_color" : "lavender"
                          };

                          // if you were to bind the above object
                          // as a FormCollection by doing the following
                          var example = new Mustang(".some-form");
                          var result = example.bindForm(data);

By passing the data variable, which is a JSON object in essence to the bindForm constructor, MustangJS does a little magic. The input with name="name", it's value is set to "Colby Perriwinkle". The input who's name is "age" has it's value automatically set to "120". Regardless of the type of the input, the correct value is used as the initial model for each Relationship. Cool huh?

Methods Reference

@return
